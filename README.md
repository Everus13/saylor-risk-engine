# MSTR-BTC Impact & Execution Engine (Strategy.com Style)

Система предиктивной аналитики, стресс-тестирования баланса и оптимального исполнения (Optimal Execution) для анализа долговой структуры MicroStrategy (MSTR) и интеллектуального управления продажами биткоина. 

Проект совмещает классические модели рыночного микроструктурного воздействия (Market Impact), сценарное квантильное прогнозирование и алгоритмы глубокого обучения с подкреплением (Deep Reinforcement Learning, PPO).

---

## 👥 Роль сооснователя (Co-founder / Product Owner)
Вся концептуальная архитектура, финансовая логика, спецификации стакана и правила риск-менеджмента спроектированы и курируются **сооснователем (Co-founder) проекта**:
* **Идея и экономическая модель:** Разработка концепции динамического триггера вынужденной продажи BTC на основе схлопывания рыночной премии к стоимости чистых активов акций MSTR (mNAV Model).
* **Финансовый консалтинг:** Интеграция структуры долговых обязательств (convertible notes), купонных выплат и лимитов краткосрочной ликвидности из реальных отчетов SEC EDGAR.
* **Трейдинг-алгоритмика:** Проектирование симулятора L2-стакана, правил OTC-маршрутизации (OTC executions), постоянного воздействия на цену (permanent market impact) и динамического расширения ликвидности для предотвращения дефицита заявок.

## 🛠 Технологический стек

* **Бэкенд (FastAPI + Python 3.10+)**:
  * **Reinforcement Learning:** Gymnasium, Stable-Baselines3 (PPO с кастомной стабилизацией: линейный LR-планировщик, энтропийная регуляризация, сжатый клиппинг).
  * **Machine Learning:** Scikit-Learn (Random Forest), LightGBM (бустинг для классификации сжатия mNAV), PyTorch (Deep MLP для предсказания квантильного импакта).
  * **Data & API:** Yahoo Finance API (история 5 лет), SEC EDGAR API (парсинг отчетов 10-Q/10-K), SQLite (персистентная база настроек и логов), Uvicorn.
* **Фронтенд (React + Vite + HTML5/CSS3)**:
  * **Стилизация:** Премиальная темная дизайн-система в стиле **Strategy.com** (высококонтрастный черный фон, неоновый оранжевый Bitcoin-акцент, моноширинные числовые индикаторы).
  * **Визуализация:** Recharts (интерактивные графики инвентаря, объемов и структуры стакана), Lucide React.

## 🚀 Подробное руководство по работе с платформой

Панель управления разделена на 4 ключевые функциональные зоны (вкладки):

### Вкладка 1: Финансовые Обязательства MSTR (SEC)
Служит для стресс-тестирования баланса компании и оценки дефицита ликвидности.
1. **Настройки симуляции (Левая панель)**:
   * **Период прогнозирования:** Выберите горизонт планирования (30, 90, 180 или 360 дней).
   * **Резервы кэша (MSTR):** Установите текущий запас ликвидности вручную или используйте спарсенные данные.
2. **Аналитические метрики (Центр)**:
   * Отображает общие обязательства к выплате, чистый дефицит кэша в USD и итоговый объем BTC, который MSTR **вынуждена** будет продать на рынке для покрытия долгов в случае закрытия ATM-программ финансирования.
3. **Графики и таблицы выплат**:
   * В реальном времени парсит базу SEC EDGAR и выводит интерактивный график выплат по облигациям и купонам с разбивкой по конкретным датам и типам инструментов.

### Вкладка 2: Симулятор Импакта L2
Позволяет протестировать мгновенное влияние крупной сделки на стакан заявок.
1. **Запуск ордера**:
   * Задайте объем продажи в BTC (например, 150 BTC) и выберите источник стакана: **Бинанс (синтетический)** или **CCXT Binance Live L2 (реальный)**.
2. **Сценарный анализ (Квантили)**:
   * Модель **Deep MLP (PyTorch)** рассчитывает вероятное падение цены BTC для трех сценариев: оптимистичного (квантиль 10%), медианного (квантиль 50%) и пессимистичного (квантиль 90%).
3. **Глубина стакана**:
   * Интерактивная тепловая карта (AreaChart) визуализирует плотность Bids/Asks и точку прохода ликвидности (L2 Walk).

### Вкладка 3: Оптимальное Исполнение (RL)
Главный инструмент квант-трейдера для оптимизации крупномасштабных продаж.
1. **Параметры симуляции**:
   * Задайте общий объем продажи, количество шагов (временных интервалов), ликвидность стакана и долю OTC-исполнения.
   * Выберите стратегию: **Равномерное капание (TWAP)** или **Динамическая оптимизация (RL PPO Агент)**.
   * Для RL-агента выберите тип: **Базовый** (синтетика), **Стресс** (низкая ликвидность) или **Адаптивный** (обученный на 2-летней реальной истории BTC с динамической плотностью стакана).
2. **Сравнение результатов**:
   * График инвентаря в реальном времени сопоставляет траекторию RL-агента с TWAP.
   * Выводится чистая выручка, средневзвешенная цена выполнения (VWAP) и полученная премия/дисконт к TWAP.
3. **Калибровка агента (Web Console)**:
   * Вы можете переобучить любого агента "на лету". Выберите тип агента, количество шагов обучения и нажмите **«Запустить калибровку агента»**. Логи обучения будут стримиться прямо в терминальное окно через WebSockets.

### Вкладка 4: Анализ mNAV и Прогнозные Модели
Зона мониторинга системных рисков MSTR и калибровки макро-модели.
1. **Премия/Дисконт к СЧА (mNAV)**:
   * Отображает текущую справедливую стоимость чистых активов акций MSTR на основе баланса биткоинов и долгов. Выводится текущая премия к mNAV и оценка угрозы сворачивания ATM-каналов ликвидности.
2. ** ML-Угроза вынужденной продажи BTC**:
   * Показывает сглаженную вероятность вынужденного сброса BTC в течение 30 дней. Вероятность рассчитывается моделью классификации на основе макро-метрик (VIX, DXY, доходность трежерис TNX) и масштабируется по фактору **Ликвидационной срочности (Liquidity Urgency)** на базе краткосрочного долга.
3. **Исторический Бэктестинг**:
   * Выводит сравнительную статистику (Accuracy и F1-Score) моделей **Random Forest** и **LightGBM** на исторических 5-летних данных. Кнопка **«Запустить тестирование и перекалибровку»** запускает автоматический пересчет и калибрует модель-чемпион.

## 🚀 Инструкция по запуску

### 1. Запуск Backend (FastAPI)
Из корневой папки проекта активируйте виртуальное окружение и запустите сервер Uvicorn:
```bash
.\venv\Scripts\activate
python -m uvicorn backend.main:app --reload --host 127.0.0.1 --port 8000
```
API будет доступно по адресу `http://127.0.0.1:8000`, интерактивная документация Swagger — `http://127.0.0.1:8000/docs`.

### 2. Запуск Frontend (React)
Перейдите в папку `frontend`, установите зависимости и запустите Vite-сервер:
```bash
cd frontend
npm install
npm run dev -- --host 127.0.0.1 --port 5173
```
Интерфейс откроется в вашем браузере по адресу `http://127.0.0.1:5173/`.
